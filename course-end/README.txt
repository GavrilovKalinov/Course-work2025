Данный вариант контейнера довольно сырой, и в данном варианте не располагает к тому,
чтобы из него можно было достать установленные файлы.
Однако если доработать данный момент при помощи дополнительных настроек и тестов,
то получится создать крайне компактный, и при этом кроссплатформенный
торрент-клиент
Хоть в exe версии и придется меньше заморочится, зато в таком варианте
нужные файлы можно будет скачивать хоть через флешку с выводом на телефон.

docker run -e DISPLAY=$DISPLAY 
-v /tmp/.X11-unix:/tmp/.X11-unix 
-v ~/.Xauthority:/root/.Xauthority 
--net=host course:latest - команда для запуска контейнера на системах Linux

(P.S.: для других систем понадобятся иные команды для запуска)

UPD 21.05.25 : получилось правильно прокинуть томы для работы с контейнером
xhost +local:root - для авторизации X11 для локали
docker run -e DISPLAY=$DISPLAY 
-v /tmp/.X11-unix:/tmp/.X11-unix 
-v ~/.Xauthority:/root/.Xauthority
-v ~/Downloads/torrents:/app/torrents
-v ~/Downloads/bittorrent:/app/downloads
